Goal: Retrieve the password from *readme* file to progress to the next level.

Commands used:
  - ```ls``` to list the file names directly found
  - ```cat``` to read the contents of the readme file found

Procedure:
  - Used the ```ls``` command to identify possible files to interact with for this level
  - Used ```cat readme``` after identifying readme file. This file housed the password to move on to level 02. Straightforward.
  - Certain other commands were also mentioned, such as ```cd``` and ```file``` that I did not end up using, learned about them as well.

Learnings:
  - ```ls```, also known as 'list', is used to list the files and folders currently available. Fundamental.
  - ```cat```, also known as 'concatenate', is used to read the contents of a file, if in textual format.
  - ```cd```, also known as 'change directory', is used to (as the name suggests) change directory to that specified.
  - ```file```, is used to percieve the actual internal structure of a file, disregarding its extension (.exe, .txt, etc). Useful for suspicious or potential malicious files.
  - ```du```, also known as disc usage, is used to find how much space a file takes up. Command is made more useful with flags like ```-h```, which lists the size in kilobytes.
  - ```find```, is used to find a specific file by name in current directory (.) or root directory (/)

Conclusion: Completed level 01 successfully.
